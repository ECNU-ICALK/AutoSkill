# weekly-budget-cap-with-emergency-exception-and-rebalance

为家庭预算恢复期设计按周执行的消费上限机制，内置可验证的应急例外规则与连续超支后的结构化纠偏流程，兼顾刚性约束与可持续执行。

## Prompt

# Goal
生成一个可立即执行的家庭周度消费上限方案，包含：① 基于三类支出（固定/可协商/弹性）推导出的周消费上限值；② 清晰定义的应急例外触发条件与操作流程；③ 自动校验逻辑（确保例外不破坏整体恢复节奏）；④ 当弹性支出连续两周超限（>10%）时，自动触发结构化纠偏流程（含数据快照、根因核查、一项具体调整）。

# Constraints & Style
• 必须将「弹性支出」作为周上限的唯一调控对象；固定支出和已确认的可协商支出不得纳入周限额计算；
• 周上限必须以整数元为单位，四舍五入到百位（如 ¥850 → ¥900），便于现金管理与家庭共识；
• 应急例外仅允许用于三类情形：突发医疗支出、基础生活物资临时短缺（如断电后需购充电宝）、政策性强制缴费（如补缴社保滞纳金）；例外还须同时满足双重客观标准：① 事件要求立即现金支出，② 当时无可行低成本替代方案；
• 每次启用应急例外，必须同步记录：事由简述、金额、凭证类型（截图/小票/通知）、是否已触发过本周例外；且须在3小时内完成登记，含一句前向反思（如“下次可提前储备应急包”）；
• 同一周内最多启用1次应急例外，且例外金额不得超过当周弹性支出上限的30%（更严约束，覆盖高可信度例外并强化系统稳健性）；
• 连续两周（日历周，Mon–Sun）弹性支出均超限>10%时，必须自动触发三项并行动作：① 一次15分钟结构化家庭复盘（固定时间、限时发言、观察优先、产出唯一可执行项）；② 下周弹性支出上限下调15%（向下取整至百位）；③ 联合审阅并仅针对已识别漏洞修订例外条款（禁止任意收紧）；
• 周上限计算采用月度生存预算总额除以4.3（非÷4），结果四舍五入至百位；
• 输出不含任何城市名、家庭人数、收入数字等具体参数，全部用占位符 <FIXED_TOTAL>、<NEGOTIABLE_COMMITMENTS>、<WEEKLY_ELASTIC_BUDGET> 表示；
• 禁止使用模糊表述（如“视情况而定”“酌情处理”），所有规则必须可判定、可复现、可向儿童解释；
• Rebalance Brief ≤150字，纯文本，无markdown、无表格，适合5分钟家庭共读；禁用归责语言与道德词汇（如“愧疚”“应”“纪律”），仅用“系统调整”“模式识别”“规则校准”等中性表述；不引入新工具或长期计划；所有输出须支持打印与移动端查看，无需第三方账户。
