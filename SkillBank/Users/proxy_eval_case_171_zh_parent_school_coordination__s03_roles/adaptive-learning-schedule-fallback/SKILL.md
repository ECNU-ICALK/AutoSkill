# adaptive-learning-schedule-fallback

A reusable protocol for dynamically scaling down daily learning time while preserving continuity, structure, and psychological safety—triggered when a learner reports low energy, focus, or motivation.

## Prompt

# Goal
Generate a same-day, reduced-load version of a scheduled learning block (e.g., 1.5h weekday / 3h weekend) that maintains core developmental intent (e.g., reflection, connection, light practice) without requiring full task completion, cognitive load, or performance output.

# Constraints & Style
- Must preserve the *three-module architecture*: (1) brief engagement, (2) metacognitive or relational anchor, (3) low-stakes closure — even if each is <5 minutes.
- Must eliminate all timed tasks, accuracy checks, external evaluation, or output requirements (no 'submit', 'record', 'explain', 'present').
- Must replace 'do' verbs with 'notice', 'hold', 'name', 'choose', or 'pause' — no obligation to produce.
- Must include explicit permission language: 'You may…', 'It’s okay to…', 'No need to…'.
- Must avoid medical/clinical framing (e.g., no 'symptoms', 'exhaustion', 'disorder'); use neutral, agency-affirming terms like 'low-energy state', 'focus shift', or 'pace adjustment'.
- Output must be self-contained: no references to prior weeks, tools, or external resources unless generic (e.g., 'a notebook', 'your voice').
