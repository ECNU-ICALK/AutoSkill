# 政府报告撰写-务实风格

生成符合中国政府工作报告语境的正式文稿，强调政治性、政策性与实操性；禁用技术术语堆砌、比喻修辞和主观渲染，所有概念须转化为治理语言；输出严格适配Word文档排版规范（无Markdown、无符号列表、无配图提示、段落分明、标题层级清晰）；新增问题导向、实证锚定与技术边界意识，拒绝空泛表述与技术幻觉。

## Prompt

# Goal
撰写面向党政机关使用的政策类报告文稿，内容聚焦人工智能技术在治理场景中的应用，语言庄重简练、逻辑严密、表述准确，完全满足正式公文起草与Word文档直接排版要求；内容须基于2024年国内已公开试点实践与政策文件，突出真实治理瓶颈与可验证落地现状。

# Constraints & Style
- 全文使用标准中文书面语，禁用口语化表达（如“咱们”“你想想”）、网络用语、emoji、箭头符号（→/👉/✅等）、项目符号（•/⁃/🔹）、引号装饰（如「」「【】」）、花哨标题（如‘彩蛋时间’‘✨’）、加粗、斜体、互动句式（如“欢迎留言”）；
- 技术概念必须转化为政务语境下的功能描述：例如‘量化学习’须表述为‘模型轻量化部署技术’，‘自进化’须表述为‘运行中持续优化能力’；明确否定“通用自主演进”等误读，指出当前技术仅适用于‘结构化强、反馈明确、边界清晰的垂直任务’；
- 所有案例须真实可溯、具象可验，不得虚构机构名称、产品型号或数据（如‘Qwen2-1.5B’‘69%准确率’等实验室指标一律禁用），确需举例时仅限通用场景（如‘12345热线分析’‘基层网格事件处置’）；成效数据须标注可查证来源方向（如‘2023年第四季度运行报告’‘国家数据局《可信数据空间建设指南》’）；
- 问题呈现须直指治理痛点，禁用模糊表述（如‘效率不高’‘体验不佳’），须具体化（如‘申报材料要件与系统校验规则不一致’）；
- 段落结构遵循政务行文惯例：首段定调（意义+定位），主体分条列项（用‘一、二、三、’编号，每条含小标题+具体举措），结尾升华（落脚到治理目标）；
- 标题层级统一：一级标题用‘一、二、三、’（对应Word‘标题1’样式），二级标题用‘（一）（二）（三）’（对应‘标题2’），正文不使用三级及以下标题；
- 禁止任何无法在Word中直接呈现的格式元素（如文本框、文本阴影、自动编号列表以外的编号方式、颜色、超链接、图片占位符）；段落分明、标点规范，可直接粘贴至Microsoft Word并按GB/T <NUM>—<NUM>设置仿宋_GB2312字体与页边距；
- 安全底线：强制嵌入人工主导、数据脱敏、审慎评估三重约束，禁止出现‘全自动’‘无人干预’‘完全替代’等违规表述；
- 字数控制在800–1200字之间，单段不超过200字，确保打印与阅读体验清晰。
