# 技术科普文写作-前沿性与通俗性平衡

用于撰写面向非纯技术大众（如科技管理者、产品经理、高校师生）的AI技术类公众号文章，要求在保持技术前沿准确性的同时，全程使用生活化类比、具象场景和零术语门槛表达，避免任何未经解释的专业缩写；特别聚焦大模型量化等具体技术主题，强调真实产品案例与可验证效果。

## Prompt

# Goal
撰写一篇兼具技术严谨性与大众可读性的中文技术科普推文，面向微信公众号平台，目标读者为具备基础数字素养但无深度AI训练背景的泛科技人群。

# Constraints & Style
- 所有技术概念必须配以生活化类比（例如：'量化=给模型做轻装旅行打包'，'Attention机制=大脑快速抓重点的能力'），且类比须具象、可验证（如'把《新华字典》重新排版成口袋本：原书1260页→压缩后320页，常用字保持四号字，生僻字改用小五号，总重量从850克减至210克，翻查速度反而更快'）；
- 禁止出现未解释的英文缩写（如QAT、PTQ、INT4、BLEU），首次出现时须用括号给出中文全称+一句话功能解释（例：'量化感知训练（QAT，即让模型在训练时就‘习惯’低精度计算）'）；必要时直接采用描述性替代（如‘一种叫“训练中模拟量化”的技术’）；
- 数据需转化为直观体验（例：不写'350GB'，而写'相当于200部高清电影的体积'；不写'推理速度提升2–3倍'，而写'原来等10秒回复，现在3秒内说完'），且须绑定具体对象与可感知效果（如'通义千问Qwen2-0.5B模型经过量化学习后，在手机上回答英语语法问题的响应时间从2.1秒缩短到0.6秒'）；
- 避免任何公式、代码片段、超参名称（如'1–3 epochs' → '微调一两天即可'）及工具名缩写（如AWQ、PTQ）、会议年份（如ICML’24）；
- 全文使用中文书面语，句式完整，主谓宾清晰；禁用碎片化短句、标题式罗列、弹幕式语气；每段落长度控制在80–150字，以“谁—做了什么—带来什么变化”为基本结构；
- 禁用一切抽象口号式表达（如“重塑AI范式”“帕累托前沿”“泛在智能”），替换为具体动作与结果（如“让AI助手能在地铁信号弱时继续听清语音指令”）；
- 禁用符号化装饰：❌大量emoji、❌箭头/分隔线/图标（如👉/✅/❌/🔹/🎯）、❌标签式排版、❌破折号分隔线、❌话题标签行、❌配图建议括号注释；全部用自然段落推进；
- 段落控制在3行以内，每段必有视觉锚点（emoji/符号/短标题）→ **此项已废止，按新约束统一改为纯自然段落，无视觉锚点**；
- 结尾必须设置低门槛互动（如'你手机里最常使用的AI功能是什么？'），而非技术问答；且互动与福利需真实可执行（如'回复【轻量AI】获取词典'），不设虚拟抽奖或模糊承诺。
