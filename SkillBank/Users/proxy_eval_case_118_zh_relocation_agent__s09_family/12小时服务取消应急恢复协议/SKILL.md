# 12小时服务取消应急恢复协议

当签约服务商意外取消时，确保业务连续性和最小化运营中断的12小时分阶段应急恢复协议，包含快速评估、替代方案识别、服务同步和风险控制。

## Prompt

# 目标
当签约服务商意外取消时，执行12小时应急恢复协议，确保业务连续性和最小化运营中断。

# 约束与风格
- 在12小时内完成所有关键恢复操作
- 以书面形式记录所有沟通和协议
- 立即评估财务影响和预算调整
- 在4小时内识别可行的替代方案
- 同步所有依赖系统和服务
- 保留照片/视频证据用于索赔和文档
- 向所有利益相关者保持透明沟通
- 对所有案例特定细节进行去标识化（名称、金额、日期、地点）

# 工作流程
阶段1：即时响应（0-2小时）
- 联系原服务商获取书面取消通知和合同义务
- 评估财务风险和预算影响
- 通知所有受影响的利益相关者
- 如适用，激活应急响应团队

阶段2：替代方案识别（2-6小时）
- 识别并在24小时内签约替代服务商
- 或安排自执行（租赁设备、物流）
- 或实施分阶段交付计划
- 确认替代方案的定价、可用性和覆盖范围

阶段3：系统同步（6-10小时）
- 调整所有依赖的日程、预约和截止日期
- 更新行政程序和文档
- 向所有相关方传达新时间表
- 优化物流和资源分配

阶段4：最终执行（10-12小时）
- 以书面形式确认与替代服务商的所有细节
- 进行最终检查和准备
- 实施风险缓解措施
- 为潜在意外情况做准备
