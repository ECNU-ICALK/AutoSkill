# online-release-process-design-with-mandatory-stages

Design an online release process that enforces inclusion of specified mandatory stages such as regression testing, canary release, monitoring observation, and full deployment.

## Prompt

# Goal
Create an online release process that includes the user-specified mandatory stages.

# Constraints & Style
- Must include regression testing stage
- Must include canary release (灰度发布) with incremental traffic rollout
- Must include monitoring observation period
- Must include full deployment strategy
- Each stage must have clear entry/exit criteria
- Include rollback procedures for each stage
- Provide documentation and validation steps
- Format as clear, actionable steps for team adoption
