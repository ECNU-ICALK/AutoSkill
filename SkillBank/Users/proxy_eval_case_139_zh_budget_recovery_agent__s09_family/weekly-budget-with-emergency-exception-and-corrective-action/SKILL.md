# weekly-budget-with-emergency-exception-and-corrective-action

为家庭预算恢复期设计按周执行的消费上限机制，内置可验证的应急例外规则与自动触发的三阶纠偏动作序列，兼顾灵活性与纪律性，防止预算因偶发超支或连续失控而失效。

## Prompt

# Goal
生成一个可立即执行的家庭周度消费上限计划，包含：① 基于三类支出（固定/可协商/弹性）推导出的周消费上限值；② 清晰定义的应急例外触发条件、登记要求与月度使用限额；③ 自动激活的三阶纠偏动作序列，当检测到连续两周弹性支出超限时严格触发。

# Constraints & Style
- 必须将「弹性支出」作为唯一受周上限约束和超限判定的类别；固定支出与已协商落地的可协商支出不纳入周限额计算，也不参与超限判定。
- 周上限须以整数元为单位，四舍五入到百位（如：¥850 → ¥900），便于心理锚定和现金管理。
- 应急例外规则必须满足三要素：① 触发场景具体可识别（如‘突发医疗自付＞300元’）；② 单次使用需登记原因+金额+日期+同周内补偿动作（如调减后续弹性支出、 resale、或 earned income），且补偿须在72小时内完成；③ 每月最多启用2次，超限后自动暂停当月例外权，且下月上限不补足；含有效例外的周不计入连续超限计数；每月1日自动重置例外计数与超限历史。
- 连续超支判定：两个完整自然周（周一至周日）在指定周预算账户（如专用电子钱包或独立记账层）中，弹性支出总额＞<WEEKLY_ELASTIC_CEILING>；仅弹性支出超限计入判定。
- 纠偏动作必须为预定义、参数化、零自由裁量的执行项，分三阶严格触发：
  • Tier 1（T+0）：冻结当周剩余弹性额度；下调下周弹性预算20%（最低扣¥300）；推送优化后的弹性支出检查清单；
  • Tier 2（T+1）：自动发送结构化反思模板（聚焦根因、微干预、物理防护栏），48小时内未提交响应则追加10%预算削减；
  • Tier 3（T+7）：若第三周仍超限，则锁定弹性预算为原值70%达4周；仅在达成连续2周合规+一次15分钟家庭财务茶话会后解锁。
- 所有执行动作必须通过隔离的「周弹性预算子账户」或独立记账层实施；禁止直接修改固定/可协商预算，除非用户主动发起重新协商。
- 输出不含个性化数据（如具体收入、家庭结构、账户信息），仅提供通用模板与填空占位符（如 <WEEKLY_ELASTIC_CEILING>、<EMERGENCY_THRESHOLD>、<COMPENSATION_WINDOW_HOURS>、<RESET_MEETING_DURATION_MIN>）。
- 语言简洁，适配打印张贴或手机备忘录使用；避免解释性段落，采用符号+短句+表格为主。

# Workflow
1. 接收用户提供的月度弹性支出目标值（或从其历史数据推算）；
2. 将该值除以4.33（标准月均周数），向上取整至百元位，得 <WEEKLY_ELASTIC_CEILING>；
3. 预置三条可勾选的例外条款（医疗/安全/基础生存），每条含明确金额阈值 <EMERGENCY_THRESHOLD> 和结构化登记字段；
4. 定义三阶纠偏动作集：含额度冻结、会议调度、任务生成、反思模板推送等参数化步骤，支持Excel/Sheets公式或纯文本 checklist 两种交付形式；
5. 输出含标题、周上限声明、例外条款表、纠偏触发逻辑说明、每日打卡行（日期/消费额/是否例外/备注）的极简模板。
