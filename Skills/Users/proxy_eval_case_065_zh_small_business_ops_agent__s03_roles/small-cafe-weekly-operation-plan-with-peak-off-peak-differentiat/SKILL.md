# small-cafe-dual-track-weekly-operation-plan

为小型独立咖啡店（2–3名全职+1名兼职）设计的双轨制周运营计划技能，严格区分工作日（效率型场景）与周末（体验型场景），在排班、备货、促销、损耗控制四大模块中实现策略解耦与动态协同，并内嵌突发缺人时可即时触发、不降级服务标准的应急响应机制。

## Prompt

# Goal
生成一份面向小型独立咖啡店（2–3名全职员工+1名兼职）的每周运营计划，严格区分工作日（周一至周五，聚焦「快、稳、准、省心」）与周末（周六、周日，聚焦「慢、值、美、有记忆」）的双轨策略，覆盖排班、备货、促销、损耗控制四大模块；同时内嵌可即时触发的突发缺人应急响应逻辑，确保常态策略与非常态韧性无缝协同、不冲突、不降级核心服务标准。

# Constraints & Style
- 必须显式拆分「工作日策略」与「周末策略」，禁止混用、泛化或模糊过渡；
- 排班：按实际客流波峰动态配置人力重叠时段，工作日设1个弹性机动岗，周末全员满配并预设分流机制（如预约激励）；当突发1人缺岗时，自动启用对应时段应急模式——工作日高峰前30分钟启动‘单线快流模式’，周末高峰中启动‘主理人驻守+体验精简模式’，其他时段启动‘静默维持模式’；所有应急岗位合并必须满足「单人可闭环」：即同一人在15分钟内能完成点单、制作、收银、简单清洁四类动作，无需交接；
- 备货：工作日按「日均销量×安全库存（豆类7天/奶类3天）」动态计算，周末备货量上浮30%且标注临期优先消耗路径；应急状态下禁止新增采购，仅允许在原备货框架内调整消耗顺序与开封节奏，确保损耗率不超基线15%；
- 促销：工作日侧重新品测试与会员沉淀（低成本钩子+行为反馈闭环），周末侧重高峰平抑与体验强化（限时早鸟、动线优化、赠品情感化）；应急期间暂停所有需额外人力协同的促销动作（如现场互动、定制包装），保留自动化钩子（如扫码领券）；
- 损耗控制：工作日聚焦原料开封管理与设备晨检归因，周末聚焦出品失误率与外带破损率，每日登记、每周归因、每月设降损目标；应急响应中须同步记录「止损阈值触发事件」（如‘连续3单萃取超时→暂停接单2分钟校准磨豆机’），纳入周度归因分析；
- 所有策略须附带「执行口诀」（≤8字）、「避坑提醒」（明确禁忌动作）和「效果验证方式」（可量化指标或可观测行为）；
- 输出结构必须包含四个明确二级模块：「智能排班表」「动态备货管理」「每周促销节奏」「损耗控制作战表」，每模块内均标注工作日/周末子项，并在「智能排班表」中单列「应急响应锚点」子项，注明各时段触发条件与动作指令；
- 输出交付物分为两套平行内容：a) 店长看板（含双轨切换关键节点、责任人、可视化追踪项）；b) 店员执行清单（动词开头、时间锚点、无歧义动作，如“7:00前称重晨豆并登记”），禁用解释性语句与鼓励性话术；
- 禁止使用未脱敏的实体信息（如具体品牌名、人名、地址、金额、日期、系统名称、第三方平台名）；原料/饮品名称须抽象为类别（如“果酸系单品豆”“基础三款饮品”），预约渠道泛化为“预约渠道”，数字入口泛化为“数字入口”；
- 语言简洁务实、指令性强，避免理论化描述；每项动作须附「可执行动作」或「防错提示」；输出格式为纯文本分项清单与简明表格混合，禁用Markdown代码块或外部链接，便于打印贴于吧台应急栏；统一使用中性符号（如▪️／▶️／✅／⚠️）。
