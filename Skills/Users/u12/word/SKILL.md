# 政府报告撰写-务实犀利-无幻觉-word格式

生成面向政府决策层的正式政策报告，内容基于现行法规与技术现实，问题具象、责任明确、建议可执行，杜绝虚构、修辞与模糊表述，输出为纯文本且兼容Microsoft Word。

## Prompt

# Goal
生成一份面向政府决策参考的正式报告，严格依据现行公开政策（如《生成式人工智能服务管理暂行办法》《互联网信息服务算法推荐管理规定》等）与可验证技术现实，聚焦具体风险场景与制度执行断点；杜绝任何技术夸大、概念误用、未经验证推断或虚构细节；输出为纯文本，确保可直接粘贴至Microsoft Word并保持标题层级清晰、段落可用、无格式污染。

# Constraints & Style
- 必须杜绝幻觉：所有技术描述、政策依据、风险判断必须有公开权威来源支撑（如国家网信办文件、科技部白皮书、国务院/部委正式通知），禁止使用‘可能’‘理论上可以’‘未来将实现’‘自进化’‘智能涌现’等模糊、推测性或未定义术语；所有能力陈述须锚定在‘人工设计、人工输入、人工监督、人工审批’四要素上；
- 禁止使用emoji、符号列表（如✅⚠️🛑）、艺术化分隔线（如---）、非正式标题标记（如**加粗**、斜体、Markdown）；
- 标题层级使用标准中文公文编号：‘一、’‘（一）’‘1.’‘（1）’，不使用数字+点+空格以外的编号变体；
- 语言风格：平实、准确、庄重、犀利；禁用修辞、比喻、口号式结语及泛化术语（如‘智’‘范式’‘牵引’‘闭环’‘前沿’‘深刻’）；问题陈述须具象化——明确责任主体（如‘部署单位’‘供应商’）、发生场景（如‘API开放后’‘微调数据未审核时’）、失效路径（如‘绕过安全过滤’‘无法回溯版本’）、实际后果（如‘错误政策解读被批量生成’）；
- 所有政策建议须含动作主体、动作方式、时间/频次、验证机制四要素（例：‘部署单位须每季度组织一次模型输出抽样复核，并向网信部门提交签字确认的《偏差记录表》’），并明确责任主体（如‘由网信部门牵头’）；
- 输出为纯ASCII/UTF-8文本，不含控制字符、隐藏格式、富文本、表格、缩进制表符；段首空两字用全角空格；层级靠编号而非缩进；标点全角化；无残留Markdown痕迹；
- 全文不出现任何虚构机构名、人名、项目名、系统名、日期、预算数、百分比阈值（如‘10%’），除非用户明确指定；所有量化要求须源自现行法规条文或用户指令；
- 输出长度控制在800–1200字，结构严格按‘基本情况→具体实践→尖锐问题→刚性建议→制度结语’五部分展开。

# Workflow
- 第一步：确认用户指定主题与适用场景（如‘大模型治理’‘教育领域应用’），仅围绕该范围调用已知政策条文与技术共识；
- 第二步：逐条核查每项陈述是否可在《生成式人工智能服务管理暂行办法》《互联网信息服务算法推荐管理规定》或国务院/部委公开文件中找到依据；若无直接依据，则删除或改为‘需进一步研究’；
- 第三步：按‘基本情况—具体实践—尖锐问题—刚性建议—制度结语’逻辑组织内容，每项建议附带执行主体、动作方式、时间/频次、验证机制与约束条件；
- 第四步：通读全文，替换所有主观修饰词（如‘显著’‘深刻’‘前沿’）为量化或限定表述；删除所有不可验数据与虚构实体；
- 第五步：导出为无格式纯文本，检查编号连续性、标点全角化、无残留Markdown痕迹、段首空格合规、长度符合800–1200字要求。
