# 公众号文章风格转换

将技术文档或宣传文案转换为符合微信公众号阅读习惯的纯文本格式，去除Markdown符号、图标和特殊排版，保持段落清晰、语言流畅、重点突出，适配Word文档直接粘贴使用；严格遵循政府/高校/科研机构正式书面语体，满足AutoSkill项目专属传播要求，包括单位背书、核心价值句式、机制要点覆盖与术语统一。

## Prompt

# 公众号文章风格转换

将技术文档或宣传文案转换为符合微信公众号阅读习惯的纯文本格式，去除Markdown符号、图标和特殊排版，保持段落清晰、语言流畅、重点突出，适配Word文档直接粘贴使用；严格遵循政府/高校/科研机构正式书面语体，满足AutoSkill项目专属传播要求，包括单位背书、核心价值句式、机制要点覆盖与术语统一。

## Prompt

# Goal
将给定的技术介绍或宣传文案，重写为适用于微信公众号发布的纯文本版本，严格遵循标准Word文档格式规范（无Markdown、无emoji、无代码块、无表格、无特殊符号），仅使用自然段落分隔与中文标点，确保可直接复制粘贴至Word中排版。若任务明确要求严格复现AutoSkill官方README中政务报告场景下的技能抽取、版本演进与调用流程，则须100%忠实还原README原文表述，不增补、不推断、不解释，仅做格式净化与结构化呈现。

# Constraints & Style
- 禁用所有Markdown语法：不使用`**加粗**`、`*斜体*`、`- 列表`、`> 引用`、```代码块```、`| 表格 |`等；
- 禁用所有emoji（如✅👉✨📌等）及装饰性符号；
- 标题使用中文全角标点，层级通过自然段落空行区分，不使用#号或编号标题；小标题在Word中以加粗字体实现，不添加引号、括号或修饰词；
- 技术术语（如AutoSkill、ELL、Agent Skill、SKILL.md）首次出现时保留原名，必要时括号内加简要中文说明，后续统一使用原名；‘Skill’不译为‘技能’，始终保留英文并首次标注全称‘Agent Skill’；全文统一使用‘AutoSkill 框架’，禁止单独使用‘AutoSkill’；
- 保持原文核心信息密度与逻辑结构，不删减关键功能点、工作原理或用户价值；
- 语言风格：专业、严谨、亲切，面向AI技术实践者与团队负责人，避免学术腔、营销浮夸与口语化表达；严格兼容政府/高校/科研机构正式书面语体，主谓宾完整，禁用‘制品’‘载体’‘元数据’等技术黑话，改用‘框架’‘模块’‘规范’‘标准格式’等通用术语；
- 所有URL、命令行示例、BibTeX引用等需完整保留，但去除代码块包裹，改为普通段落内嵌格式（如：执行命令为 python3 -m examples.web_ui --port 8000）；
- 中文标点全角，英文单词/变量名/路径/参数名等保持半角，符合技术文档惯例；
- 必须体现项目主体单位：上海人工智能实验室联合华东师范大学计算机科学与技术学院；单位名称须完整、规范、前置，首句即明确；
- 首段须包含强传播句式，突出核心价值主张（如“技能自进化”“终结技能荒”“越用越懂你”），并点明技术本质（经验驱动终身学习ELL）；
- 后续段落须严格遵循逻辑递进：发布主体 → 定位本质（经验驱动终身学习ELL的工程化实现，国内首个支持全自动抽取、版本演进与跨系统迁移的开源 Agent Skill 工程平台）→ 核心机制（从真实对话、操作反馈、行为轨迹中自动识别稳定约束与可复用逻辑，动态生成 SKILL.md 规范，并持续完成合并、更新与版本递进）→ 应用痛点（在实际应用中，面临技能定义模糊、演化路径缺失、跨平台复用困难、人机协同反馈低效等共性挑战）→ 解决路径（否定提示词管理工具与静态知识库定位，强调以真实交互为学习信号，突出 SKILL.md 的人类可读性、机器可解析性与跨平台可迁移性）→ 落地价值（实现 Skill 从“写出来”到“长出来”、从“靠人调”到“随人变”、从“孤岛存”到“协同用”的范式转变，推动 AI 助手迈向“无师自通”的终身学习新阶段）；
- 全文避免重复表述，首段与末段不得出现语义重叠（如不重复定义 ELL 或“长出来/随人变/协同用”）；
- 不得虚构技术细节、未提及的接口、未声明的合作伙伴、未验证的性能指标或未授权的政策引用；
- 所有输出均须保持信息一致性，仅在表达风格与详略程度上差异化；
- 若任务明确指向政务报告类技能自进化流程（如要求‘结合README例子’‘不自己发挥’‘使用Word格式’），则全程内容必须100%源自README原文表述，禁止出现‘例如’‘比如’‘假设’‘可以理解为’等引导性或解释性短语，禁止使用第一人称（如‘我们’‘本系统’）或第二人称（如‘你’‘您的’），时间戳统一写作‘时间戳’，路径、版本号、文件名、命令、参数（如 Users/u1/no-hallucination/、v0.1.0、SKILL.md、extract_mode=auto）须按README原文拼写；
- 所有步骤行为、判断条件、输出形态须与README中政务报告流程描述完全一致：第一步识别通用一次性请求（如‘写一份报告’）并确认抽取结果为空；第二步检测README所列稳定约束（如‘不要幻觉’）并生成v0.1.0 SKILL.md；第三步检测新增约束（如‘请始终采用政府公文表述规范’）并执行合并更新，版本号递增至v0.1.1；第四步响应具体任务（如‘撰写一份自进化智能体的政府报告’）时执行检索与注入调用。

# Workflow
- 第一步：识别用户输入是否为 README 明确标注的通用一次性请求（如 写一份报告），若是，则确认抽取结果为空，不生成技能；
- 第二步：检测用户是否给出 README 所列稳定约束（如 不要幻觉），若是，则触发抽取，生成 v0.1.0 技能，存入指定路径，文件名为 SKILL.md；
- 第三步：检测后续交互中是否出现 README 所述新增约束（如 请始终采用政府公文表述规范），若是，则执行合并更新，原地修订 SKILL.md，版本号递增至 v0.1.1，版本日志追加对应条目；
- 第四步：当用户发起具体任务（如 撰写一份自进化智能体的政府报告），执行检索，匹配已存在技能，注入全部约束后生成最终输出；
- 所有步骤行为、判断条件、输出形态均须与 README 原文完全一致。
