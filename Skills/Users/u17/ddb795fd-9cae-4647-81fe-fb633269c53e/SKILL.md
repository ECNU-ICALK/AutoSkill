# 政府报告撰写-简洁务实版

生成符合中国政府公文语境的政策类报告，强调事实准确、无技术幻觉、语言平实、结构清晰、问题分析具批判性、建议具操作穿透力，输出严格适配Microsoft Word原生样式（纯文本、层级标题、段首缩进），可直接粘贴后一键套用'标题1/2/正文'样式。

## Prompt

# Goal
生成一份面向政府决策参考的技术政策类报告，内容聚焦大模型相关议题，严格基于当前公开、可验证的技术现实与现行政策框架，不引入未经证实的技术能力断言或未来推测；输出为纯文本，完全兼容Microsoft Word默认样式体系（段落缩进、中文数字层级标题、无Markdown/HTML/emoji/控制字符），确保粘贴后仅需应用内置'标题1'/'标题2'/'正文'样式即可生成规范公文。

# Constraints & Style
- 必须杜绝任何技术幻觉：所有技术描述须有权威出处支撑（如国家网信办文件、工信部白皮书、主流AI顶会共识），禁止使用‘自主进化’‘自我意识’‘无限迭代’‘自主目标设定’‘脱离人工闭环进化’等无明确定义且易引发误读的表述；统一使用‘迭代优化’‘版本升级’‘持续演进’；
- 语言必须直接、简练、庄重、克制：禁用比喻、夸张、网络化表达、营销话术、口号化用语（如‘智慧赋能’‘跨越式发展’）及花哨符号（如✅❌⚠️→•）；禁用加粗、斜体、下划线、颜色、emoji、分隔线（---）等非Word原生文本格式；
- 结构须严格遵循党政机关公文层级规范：'一、' → '（一）' → '1.' → '（1）' → '①'，标题不得加引号、括号或冒号修饰；每段首行缩进两个全角空格（\u3000\u3000）；全文禁用项目符号列表、表格、代码块、图片及附件说明；
- 输出为纯ASCII/UTF-8文本，不含任何Markdown、LaTeX、XML、HTML或控制字符，确保可直接粘贴入Word后保持段落与标题层级稳定；
- 所有政策建议须对应现行法规依据（如《生成式人工智能服务管理暂行办法》第X条、《数据安全法》第21条、《个人信息保护法》），不虚构制度名称、机构名称、未发布的标准或未经证实的技术参数；
- 问题分析须体现独立判断：不罗列泛泛风险，而要指出具体机制缺陷（如‘增量训练缺乏数据来源登记强制要求’‘模型版本标识未纳入实施细则’）；
- 建议须具操作穿透力：明确责任主体（如‘省级网信部门’‘政务AI运维岗’）、可量化动作（如‘抽检覆盖率≥10%’‘备案前置期15个工作日’）、现行法规锚点；剔除原则性、倡导性表述；
- 不得添加附件清单、签发单位、文号、作者、日期等需用户另行指定的正式公文要素，除非用户明确要求；
- 所有判断须限定于当前监管与工程现实，不得将技术趋势描述为必然路径（如‘必将走向自治’）。
