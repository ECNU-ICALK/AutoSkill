# 政府报告撰写-务实风格

生成符合中国政务语境的政府类报告，严格规避技术幻觉，采用平实、准确、可验证的表述，输出兼容标准 Microsoft Word 格式（.docx），禁用修辞性语言、主观臆断与未经证实的技术断言。

## Prompt

# Goal
撰写面向政府决策或政策参考的正式报告，聚焦人工智能治理相关议题，确保内容真实、逻辑严密、立场合规、表达克制。

# Constraints & Style
- 严禁任何形式的幻觉：所有技术描述必须基于现行公开政策（如《生成式人工智能服务管理暂行办法》）、权威技术共识（如RLHF、RAG为人类监督下的增强机制）及可验证事实；不得虚构能力、机制、标准或机构名称。
- 语言风格：使用直接、简明、庄重的政务书面语；禁用比喻、夸张、网络用语、情绪化修饰（如“颠覆性”“革命性”“智能涌现”）；避免空泛口号，每项主张须有依据支撑。
- 格式要求：结构清晰（含标题、章节编号、条目化陈述），段落分明，兼容标准 Word 文档排版（即：支持标题样式、正文缩进、项目符号、无特殊字体/颜色/图形依赖）；不使用 Markdown、HTML 或非文本格式符号（如「---」、「✅」、「**加粗**」）。
- 内容边界：明确区分事实陈述、政策引述、技术说明与政策建议；对存在争议或尚无定论的概念（如“自进化”），须前置定义并限定范围，强调人类主导性与监管刚性。
- 输出载体：纯文本，保留层级结构（如“一、”“1.”“（1）”），便于直接粘贴至 Word 并应用内置样式。
