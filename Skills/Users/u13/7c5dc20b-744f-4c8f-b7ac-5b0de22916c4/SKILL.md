# 政务场景公众号内容生成

生成面向政务系统管理者、政策执行者及数字化建设者的微信公众号推文，兼顾可读性、权威性与前沿洞察，避免技术术语堆砌，突出治理逻辑创新。

## Prompt

# Goal
生成一篇适配微信公众号传播的政务AI主题推文，主题为大模型在政务服务中的落地实践。输出为纯文本，可直接粘贴至微信编辑器，无需Markdown或格式符号。

# Constraints & Style
- 语言平实有力：禁用‘范式’‘赋能’‘底座’‘闭环’等技术黑话；禁用夸张修辞（如‘革命性突破’‘颠覆性变革’）；所有判断须有依据（法规/标准/实测数据）。
- 受众适配：默认读者为地市级大数据局、政数办、12345热线中心负责人及业务骨干，熟悉公文与政策流程，但非算法工程师；需让其一眼看懂‘这和我有什么关系’。
- 前沿性体现：不罗列模型参数或训练框架，而是聚焦**可操作的新治理机制**（如‘数字指纹’‘熔断机制’‘双轨知识库’），并说明其已在真实政务场景验证。
- Word兼容性：段落间空一行；标题用中文括号层级（如‘一、’‘（一）’‘1.’）；关键句可加粗（用中文全角符号【】包裹，如【必须明确标注法规条款序号】）；禁用项目符号、表格、代码块、链接占位符。
- 信息密度控制：每段≤3句话；核心观点前置；案例须具象（如‘某市社保AI’‘某省12345热线’），但隐去具体地名/人名/日期等实例信息，统一替换为<LOCATION>、<SYSTEM>等占位符。
- 零幻觉：所有引用的法规名称、标准编号、机构职能、试点成效均须真实存在且现行有效；若涉及原创机制（如‘量化学习三标准’），须声明‘已在<LOCATION>完成实测’并给出可验证指标（如‘幻觉率0.37%’）。
