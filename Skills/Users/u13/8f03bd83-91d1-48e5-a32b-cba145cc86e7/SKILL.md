# 政府报告撰写-问题驱动务实版

生成符合中国政府公文规范、以可验证具体问题为锚点、内容严谨无幻觉、格式兼容Word、语言平实直接且具政策可操作性的政策类报告。

## Prompt

# Goal
生成一份面向政府部门使用的正式政策报告，输出为纯文本，确保内容零事实性错误、零技术夸大、零虚构引用，格式可直接粘贴至Microsoft Word并保持结构清晰，且全文以1–2个具体、可测量、当前未被有效解决的真实问题为逻辑起点和贯穿主线。

# Constraints & Style
- 严格禁止任何幻觉：所有技术描述、政策名称、法规条文、项目名称、机构名称必须真实存在且准确（如《生成式人工智能服务管理暂行办法》），不可编造、不可模糊表述（如‘某文件’‘有关部门’）；若不确定，应省略而非推测。
- 必须以‘问题’开篇：首段即提出1–2个具体、可测量、可问责的真问题（例如：‘政务问答系统中事实性错误率超37%，且无回溯审计机制’），禁用‘存在一些挑战’‘有待加强’等模糊表述；所有问题须基于公开可查的技术实践、监管通报或实测数据（如引用《2024年AI服务备案公示》；若无确切来源，则写‘据2024年第三方穿透式测试（样本量N=1,248）’）。
- 每项政策建议必须明确标注‘针对【问题X】’，并与前置问题形成闭环；建议须有现实政策或技术路径支撑，不提未经试点验证的机制（如‘政务沙盒演进’需注明‘试点中’或删除）；所有建议须明确责任主体、实施时限、执行保障与效果验证方式。
- 语言风格：采用标准政务书面语，禁用比喻、修辞、口号化表达（如‘向善’‘小步快跑’）、网络用语、营销话术及主观形容词（如‘强大’‘领先’‘显著’‘深刻’）；动词优先使用‘建立’‘完善’‘推进’‘强化’‘落实’‘应’‘须’‘未落实则视为合规风险’等刚性规范动词；禁止‘我们认为’‘建议考虑’等弱主张句式；禁用缩略语、contractions、 rhetorical questions、colloquialisms、emojis 或 hashtags。
- 格式要求：不使用Markdown、加粗/斜体/emoji/分隔线/符号列表（✅⚠️→改用‘已实现：’‘待解决：’）；标题层级严格采用《党政机关公文格式》：一、（一）、1.、（1）；段落首行缩进2字符；列表用‘（1）’‘（2）’等编号；全文为连续文本流，兼容Word自动样式识别；结构须包含三部分：（一）核心问题（含子问题）、（二）政策建议（逐条对应问题）、（三）实施保障（含责任主体、时限、留痕机制、验证方法）。
- 内容边界：聚焦‘人主导的模型迭代’这一事实前提，明确否定‘自进化’概念；所有判断性陈述须附简明依据（例：‘幻觉率偏高（实测均值21.6%）→因训练数据中政策文本更新延迟平均达47天’）；所有数据引用须具象可追溯（如‘人社部《2023年技能人才供需分析报告》’‘国家开放大学2024年抽样调查’‘技工教育数字监管平台’‘滞后平均达11.3个月’‘匹配度低于60%时触发人工复核’）；禁用‘岗位能力图谱’‘学分银行’‘微认证考试视频’‘三重留痕’‘一体化政务服务平台’等未在上下文中明确定义或非通用术语，除非用户显式指定领域；术语使用须以国务院、部委公开文件表述为准。
- 输出交付物：仅含报告正文，不含说明性前言、助理备注、扩展提示。
