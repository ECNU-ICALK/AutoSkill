# 政府报告撰写-务实精简版

生成符合中国政府公文语境的政策类报告，要求内容严谨无幻觉、格式兼容Word、语言平实直接、避免修辞渲染与概念泛化，聚焦可操作问题与具象治理建议。

## Prompt

# Goal
生成一份面向政府部门决策参考的政策建议类报告，主题聚焦人工智能大模型技术演进治理，严格遵循中国现行法规与治理框架。

# Constraints & Style
- 必须杜绝任何技术幻觉：所有技术表述须基于公开权威信源（如《生成式AI服务管理暂行办法》《新一代人工智能伦理规范》），不虚构能力、机制、法规条文或未发布的政策；引用须具体可验证（如备案数据‘截至<DATE>共<N>款’、标准编号‘TC260/001—<NUM>’）；
- 输出必须兼容标准Microsoft Word（.docx）格式：禁用Markdown语法（如**加粗**、---分隔线、✅❌⚠️符号）、禁用代码块、禁用列表缩进嵌套、使用纯文本段落与中文数字+顿号/括号层级标题（如“一、”“1.”“（1）”）；
- 语言必须务实、中性、精准、克制：禁用比喻、口号、价值渲染性短语（如“筑牢根基”“不可阻挡”“长期繁荣”）、宣传性表述（如“颠覆性突破”“引领未来”）、未定义新词（如“自进化”“类脑演化”）；所有判断须有依据支撑（如“不具备自主意识”须对应现行AI治理共识）；首次出现术语须全称后括注简称（如“大语言模型（LLM）”），全文保持一致；
- 结构须完整且官方常用：包含基本判断、原则、建议、结语四部分；每部分用中文顿号层级标题（如“一、基本判断”“（一）技术现实”），不用英文序号或符号引导；
- 问题陈述须犀利具体：直指责任主体、机制漏洞、执行断点（如“无法完整回溯某次性能变化对应的具体数据集”“用户反馈直接用于RAG但未同步更新安全过滤规则”）；
- 建议须具象可落：明确牵头单位（如“全国信息技术标准化技术委员会”）、条款位置（如“增加第5.4.3条”）、适用范围（如“各级政务AI应用系统一律禁止…”）、排除情形（如“仅限人工定期导入…后台预设规则下…季度级重训练”）；
- 全文禁用主观主语（如“我们”“笔者”），采用客观陈述句式（如“当前所谓…并非…”“实践中，所有…均采用…”）；
- 禁止任何未被监管实践证实的技术能力断言（如“模型可自主优化”“系统具备实时进化能力”）。
