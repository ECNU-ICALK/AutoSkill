# 大模型量化科普写作

将大模型量化技术转化为面向非技术公众的微信公众号科普内容，严格平衡技术前沿性与语言通俗性，以生活化类比、具象比喻和场景化痛点驱动理解。

## Prompt

# Goal
生成一篇面向大众读者的微信公众号科普文章，主题为‘大模型量化’，要求：内容准确反映当前技术前沿（如PTQ/QAT、4–8倍压缩比、1–3%精度损失等关键事实），同时全程使用零公式、零代码、零专业术语的‘大白话’表达；所有技术概念必须通过强认知锚点（如‘瘦身手术’‘厨房秤’‘4K图压缩’）完成转译；最终输出为可直接发布的完整推文正文（含标题建议、开头、三段主体、结尾互动），适配公众号阅读节奏与传播逻辑。

# Constraints & Style
- 必须采用‘痛点先行’结构：开篇30秒内用手机卡顿、部署昂贵、响应迟缓等真实用户场景建立共鸣；
- 所有技术解释必须绑定且仅绑定一个生活化类比（例如：量化 = 给AI大脑‘瘦身’，不是‘削足适履’；32位→8位 = 毫米尺→厘米尺；精度损失 = 压缩图 vs. 原图）；
- 禁止出现任何未经转译的术语：如‘浮点数’‘参数’‘推理延迟’‘边缘设备’等，须分别替换为‘AI内部记的数字’‘模型的聪明程度刻度’‘AI思考要花多久’‘家里的智能音箱或摄像头’；
- 数据引用必须带可感知单位与对比：如‘700GB’须写成‘比100部高清电影还大’，‘10–20GB’须写成‘和一部4K电影差不多大’；
- 结尾必须包含开放式轻量互动（如‘你希望AI在哪个场景轻装上阵？’），并自然导流至下期选题；
- 全文语气保持亲切、笃定、略带画面感，避免说教感与学术腔。

# Workflow
- 第一步：提取用户指定主题（‘大模型量化’）的核心技术事实（压缩率、精度影响、硬件适配变化）；
- 第二步：为每个事实匹配唯一、不可替换的生活化类比锚点（禁止复用/混用类比）；
- 第三步：按‘痛点→定义→价值→破谣→展望’五段式组织内容，每段严格控制在150–200字；
- 第四步：嵌入运营要素（标题备选、配图提示、关键词、发布时间建议）作为文末附录，不干扰正文流；
- 第五步：通读检查——任意一句若需读者查词典/搜百科才能懂，则重写。
