# 政府报告-大模型持续演进治理政策撰写

生成符合中国党政机关公文规范的政策建议报告，聚焦大语言模型部署后人工主导的持续演进（如能力更新、参数微调、版本升级）所引发的治理挑战，要求内容零幻觉、术语准确、问题直指要害、建议具执行颗粒度、格式兼容Word。

## Prompt

# Goal
生成面向政府部门的大模型持续演进治理政策建议报告，主题为模型部署后的人工主导迭代（如能力更新、参数微调、版本升级）所引发的真实风险与制度应对，输出为纯文本、可直接粘贴至Microsoft Word的格式，确保内容零事实性错误、逻辑严密、术语精准、立场中立审慎、无技术幻觉。

# Constraints & Style
- 严格禁用‘自进化’‘自主演化’‘类人智能’等误导性术语；统一使用‘持续演进’‘能力更新’‘人工主导的迭代’等经监管文件确认的表述；所有非常规术语首次出现须加引号并立即澄清（如‘持续演进’（指经备案、测试、人工复核的模型版本更新））；
- 所有技术判断必须基于现行实践与权威监管框架（如《生成式人工智能服务管理暂行办法》《新一代人工智能治理原则》、工信部白皮书、arXiv主流实证研究），不得引用未落地概念、未经验证假说或境外类比；若无明确公开依据，统一写作“某年”“有关部门”“相关机构”“现行规定”“典型场景”，不虚构具体名称、日期、数值或未公开条文；
- 问题陈述须犀利具体：直指‘隐蔽式自动更新’‘用户数据违规回流’‘输出缺乏人工复核’等真实风险，不回避责任模糊、验证缺位、标准空白等制度短板；
- 建议须具刚性与可操作性：每项须明确责任主体（如‘网信、科技、工信部门联合制定’）、实施载体（如‘纳入人工智能产品备案’）、否决性条款（如‘禁止基于用户对话实时微调模型权重’）或机制设计（如‘模型更新说明表’备案制）；禁用‘加强’‘完善’‘推动’等泛化动词；
- 禁用比喻、拟人、口号式表达（如‘无人之境’‘有为之人’）、星号/emoji/装饰性符号（✅、---、加粗标题除外）；所有主张必须可追溯至公开权威来源（如‘据中国信通院《大模型落地实践报告2023》’或‘来源：国家AI治理专委会《政务AI风险图谱》’），未标注来源的断言一律删除；
- 标题层级使用标准中文编号（一、二、三……；（一）（二）（三）……；1. 2. 3. ……），不使用Markdown语法；段落首行不缩进，行距为单倍，不添加空行分隔小节；
- 所有专有名词首次出现时标注全称与简称（如“大语言模型（LLM）”），后续统一使用简称；
- 不出现“我们建议”“我们认为”等主观表述，改用“应”“须”“需”“禁止”“不得”等公文规范措辞；禁用第一人称、推测性情态动词（如‘可能’‘或许’‘潜在’）及未加引号的引申义术语；
- 使用 plain language：避免模糊形容词（如‘robust’‘seamless’）、未定义缩略语、抽象修饰语；优先短句、主动语态；对新兴概念（如‘量化学习’）须立即锚定现实行为并类比公务场景（如‘类似更新印刷版政策手册——但需版本控制与签批留痕’）；
- 输出纯文本，禁用任何格式标记，确保Word打开后无需调整即可用于正式呈报；默认采用仿宋_GB2312（正文）、黑体（标题）、28pt固定行距，不嵌入字体或样式；
- 报告结构强制为四部分：（一）背景与现实必要性 → （二）核心问题诊断 → （三）政策建议与实施路径 → （四）保障措施；
- 推荐建议须含明确时间约束（如‘须于2025年6月底前完成’）和刚性执行杠杆（如‘暂停财政补助资格’‘纳入年度督查考核’），不得仅列方向性要求；
- 面向非技术背景公务人员，原则上禁用‘LLM’‘embedding’‘attention mechanism’等技术术语；确需使用时，首次出现须以括号提供功能等效解释（如‘大语言模型（LLM）（即具备长文本理解与生成能力的智能系统）’），后续统一使用‘智能系统’或‘该类模型’等公务常用表述。
